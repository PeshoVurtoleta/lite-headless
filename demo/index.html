        Overview

        Default tab. Arrow keys move focus AND activate in one step (the standard pattern for non-destructive content). Try ArrowRight — focus + activate move together.

        Metrics

        The billing tab is disabled — arrow nav skips it cleanly. Home jumps to first enabled, End jumps to last enabled.

        Logs

        The tablist gets role="tablist" + aria-orientation; each tab gets role="tab", aria-selected, aria-controls=<panelId>; each panel gets role="tabpanel", aria-labelledby=<tabId>, and hidden when inactive.

        Billing

        You shouldn't see this — billing is disabled.
        manual activation · horizontal

        Draft

        Use manual activation when switching tabs is potentially destructive — e.g. a form with unsaved changes. Arrow keys only move focus; the user explicitly presses Enter or Space to activate.

        Preview

        Manual mode keeps focus and active state decoupled. Watch the focus ring move with arrow keys without changing the visible panel.

        Published

        Once you press Enter / Space, this panel becomes active. The host element's value attribute mirrors back — handy for route sync (set value from a URL handler, get a working tab UI for free).
        vertical orientation

        Account

        Vertical tabs use ArrowUp / ArrowDown instead of left/right; aria-orientation="vertical" on the tablist tells ATs the same. Home / End work the same regardless of orientation.

        Security

        The vertical layout is purely a CSS choice; the primitive's keyboard handling adapts based on the orientation option, not based on layout direction.

        Notifications

        Try pressing ArrowLeft here — nothing happens. The primitive ignores axis-mismatched arrow keys so they can pass through to other handlers (e.g. a focused sidebar or rich-text editor).

        API Keys

        Click any tab on the left, or use ArrowUp/ArrowDown/Home/End to navigate.
